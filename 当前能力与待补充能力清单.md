# 当前能力与待补充能力清单

本文只描述仓库当前代码已经具备的能力，以及为了达到一期目标仍需补充的能力。

原则：

- 只写当前仓库已经实现或明确未实现的内容，不把规划写成现状
- 以代码主链路、接口能力和运行方式为准，不以口号或路线图替代现状
- 用于 README、简历项目介绍、阶段复盘和后续迭代排期

## 当前能力

### 1. 系统定位

当前仓库是一套面向企业内部知识库问答场景的上传式 RAG 基线实现，重点是把“文档上传入库 + 检索问答 + 工作流 + 审核 + 运维后台”串成一条可运行闭环，而不是停留在单接口 demo。

### 2. 助理管理

- 支持创建、查看、编辑、删除助理
- 支持配置助理名称、描述、系统提示词、默认模型、默认知识库和审核规则
- 支持助理版本快照、版本列表查看和历史版本恢复

当前价值：

- 系统不是单助理硬编码结构，已经具备基础配置中心能力
- 助理配置和会话、知识库之间已经建立稳定关联

### 3. 知识库管理

- 支持创建、查看、编辑、删除知识库
- 支持统计知识库下文档数量
- 支持统计知识库被多少助理绑定
- 支持知识库资源清理和关联约束校验

当前价值：

- 已经具备“多知识库”管理能力，不是单库问答 demo
- 后续可以在这个基座上继续补权限过滤和知识治理

### 4. 文档上传与入库

- 支持上传并入库以下格式：
  - `txt`
  - `md`
  - `csv`
  - `json`
  - `yaml` / `yml`
  - `xml`
  - `html`
  - `pdf`
  - `doc`
  - `docx`
- 上传后会创建异步处理任务，不阻塞前端请求
- 文档会经过解析、分块并写入 Qdrant
- 支持按知识库查看文档列表
- 支持删除文档及对应索引数据

当前价值：

- 已经覆盖企业内部知识库常见文本格式
- 入库链路具备任务化雏形，后续可以继续补幂等、失败分类和 OCR

### 5. 检索增强问答

- 支持按助理默认知识库进行问答
- 支持请求时显式指定单个或多个知识库范围
- 支持返回引用片段、文档名、分块序号和检索分数
- 支持多知识库 fan-out 检索后统一重排

当前检索基线：

- `Qdrant dense retrieval`
- `lexical rerank`
- 多知识库路由检索

边界说明：

- 当前还不是真正意义上的 hybrid retrieval
- 目前没有独立 BM25 / lexical retriever
- 目前是向量召回后，再做词面重排

### 6. LangGraph 会话工作流

- 问答链路不是简单的“检索 -> 生成”
- 当前工作流已接入 LangGraph
- 支持把会话状态、历史消息、知识库范围和审核规则一起纳入流程
- 支持工作流运行结果回写会话运行时状态

当前价值：

- 项目已经具备“可解释的流程节点”结构，后续扩展澄清、审核、拒答更自然
- 比普通 RAG demo 更适合展示工程化 workflow 设计

### 7. 意图漂移检测与澄清

- 支持检测当前提问是否偏离会话主线
- 命中后可返回澄清提示，而不是直接继续回答
- 支持维护当前会话目标、待确认问题和澄清阶段

当前价值：

- 已经体现多轮会话里的状态管理
- 不是一次性问答接口，而是带会话上下文的 RAG 工作流

### 8. 人工审核中断与恢复

- 支持按助理审核规则识别高风险问题
- 命中后可中断自动回答，转入待审核状态
- 支持生成审核任务
- 支持审核通过、驳回和人工补充答案
- 审核完成后可恢复流程并回写结果

当前价值：

- 这是项目最像企业场景的一块能力
- 适合展示“高风险问答不能直接放给模型”的系统设计意识

### 9. SSE 流式回答

- 聊天接口支持 SSE
- 支持发送 `start`、文本增量、`completed`、`error` 等事件
- 支持在流式开始时返回本轮选中的知识库和检索条数

当前价值：

- 前端交互体验更接近真实产品
- 也便于后续接入更细粒度运行事件和 trace

### 10. 会话与消息管理

- 支持创建、查看、删除会话
- 支持按助理筛选会话
- 支持查看会话消息列表
- 支持为会话维护标题、消息数和运行时状态

当前价值：

- 已经不是纯接口级 RAG，具备基础聊天产品结构

### 11. 任务中心

- 支持查看文档处理任务
- 支持按类型、状态和 SLA 状态筛选
- 支持失败任务重试
- 支持批量重试失败任务
- 支持展示任务关联的文档、知识库和目标状态

当前价值：

- 入库流程已有基本的运维视角
- 任务失败后不是只能手工重传

### 12. 审计日志

- 支持按会话查看审计日志
- 支持按审核任务查看审计日志
- 已记录普通问答、澄清、审核挂起、审核处理和失败等关键事件

当前价值：

- 提升问题追踪能力
- 便于后续做复盘和可观测性扩展

### 13. 鉴权与角色权限

- 支持登录和当前用户查询
- 支持 Bearer Token 鉴权
- 支持按接口能力做权限控制
- 当前内置管理员、运营、访客等演示角色

当前价值：

- 系统不是完全匿名访问
- 已具备后台管理系统常见的角色权限基础

### 14. 系统总览与维护入口

- 支持系统总览接口
- 支持系统维护任务入口
- 支持基础健康检查

当前价值：

- 已具备最小运维后台接口面

### 15. 工程化基础

- 前后端分离：
  - 后端为 FastAPI
  - 前端为 Vue 3 + TypeScript
- 后端具备数据库迁移能力：
  - Alembic
- 支持 Docker / Docker Compose
- 支持开发环境 SQLite
- 支持生产导向的 PostgreSQL / Postgres Checkpointer
- 已包含一组后端测试用例

当前价值：

- 项目形态已经接近“企业内部试点系统基线”
- 不是只有 notebook 或单脚本

## 当前还需要补充的能力

### 1. 真正的检索主链路升级

当前问题：

- 现有能力仍是 `dense retrieval + lexical rerank`
- 还没有真正的 lexical retrieval
- 还没有 dense / lexical 并行召回和融合

建议补充：

- 独立 BM25 或其他 lexical retriever
- dense + lexical 并行召回
- RRF 或加权融合
- 检索 trace、融合前后排序和 context assembly

为什么优先：

- 这是当前项目最大的技术短板
- 也是最影响“RAG 是否真的做扎实”的部分

### 2. 知识库级 ACL 预过滤

当前问题：

- 当前没有真正完成检索前 ACL 过滤
- 企业场景下，用户可见范围不能依赖检索后再裁剪

建议补充：

- 知识库级 ACL 模型
- 检索前可见范围过滤
- 会话侧记录本轮实际授权范围

为什么优先：

- 这是“企业级”而不是“个人 demo”的核心分界线之一

### 3. 知识治理字段与版本治理

当前问题：

- 文档治理维度仍然偏弱
- 缺少稳定的去重、版本和来源管理

建议补充：

- `source`
- `version`
- `updated_at`
- `tags`
- `content_hash`
- 文档去重策略
- 文档版本覆盖策略

为什么优先：

- 企业知识库一旦进入长期维护阶段，这些字段决定可管理性

### 4. 入库工程化能力

当前问题：

- 异步任务机制还是雏形
- 对失败分类、幂等、重试策略和大文件处理还不够完整
- PDF 解析还没有 OCR

建议补充：

- 更细粒度任务状态机
- 幂等处理
- 失败分类与错误码
- 稳定重试机制
- 扫描件 PDF OCR
- 按文档类型定制 chunking

为什么优先：

- 这会直接影响真实文档能否稳定进入知识库

### 5. 生成层可信度控制

当前问题：

- 现在有引用，但还没有结构化强校验
- 低置信度拒答能力不足

建议补充：

- 结构化回答 schema
- 引用强校验
- 证据不足时拒答
- Prompt 版本管理
- 回答可信度标记

为什么优先：

- 企业用户更在意“能不能信”，而不是“能不能答”

### 6. 评测体系

当前问题：

- 目前缺少稳定评测和回归基线
- 难以证明一次改动是变好还是变差

建议补充：

- golden dataset
- `Hit@k`
- `Recall@k`
- `MRR`
- 引用命中率
- 拒答率
- 检索 / 生成分层回归脚本

为什么优先：

- 评测是项目从“能跑”升级到“可验证”的关键一步

### 7. 可观测性与稳态运行

当前问题：

- 当前有 system overview，但还不是完整可观测性体系
- 限流、超时、降级和告警还没补齐

建议补充：

- 检索耗时、生成耗时、端到端耗时
- 外部依赖失败率
- 限流
- 超时
- 重试
- 降级策略
- readiness checklist
- 基础告警

为什么优先：

- 这是“可试点”和“可上线”之间的主要差距

## 当前不应误写成已具备的能力

以下内容如果没有补齐，不应在 README、简历或项目介绍中表述成“已完成”：

- 真正的 hybrid retrieval
- 检索前 ACL 过滤
- OCR 入库
- 文档去重和版本治理闭环
- 结构化回答强校验
- 完整回归评测体系
- 生产级限流、降级和告警
